# Эльбрус

**Эльбрус** — веб-платформа для развития сотрудников, внутренней мобильности и HR-подбора.  
Проект объединяет личный кабинет сотрудника, систему достижений, магазин бонусов, уведомления, а также HR-интерфейс для поиска и ранжирования кандидатов по требованиям к должности.

Проект разработан как выпускная квалификационная работа и демонстрирует прототип корпоративной HR-платформы с использованием Python, Gradio, MySQL, SQLAlchemy и AI-компонентов.

## Описание проекта

Цель проекта - создать единый интерфейс для сотрудников и HR-специалистов, который помогает:

- Поддерживать актуальность профиля сотрудника
- Фиксировать образование, опыт, навыки, курсы и достижения
- Мотивировать сотрудников через достижения и баллы
- Обменивать баллы на бонусы
- Уведомлять сотрудников и HR о важных событиях
- Подбирать внутренних кандидатов под требования вакансии
- Формировать анонимную рейтинговую таблицу кандидатов

Система ориентирована на сценарии внутреннего развития сотрудников и повышения прозрачности карьерных возможностей внутри компании.

## Настройка окружения `.env`

Отредактируйте файл `.env`.

Пример:

```env
APP_TITLE=Эльбрус

GRADIO_SERVER_NAME=127.0.0.1
GRADIO_SERVER_PORT=7860
GRADIO_DEBUG=True

DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=elbrus
DB_USER=root
DB_PASSWORD=your_password

UPLOADS_ROOT_DIR=uploads

HF_TOKEN=your_huggingface_token
HF_EMBEDDING_MODEL=sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
HF_RERANKER_MODEL=cross-encoder/ms-marco-MiniLM-L-6-v2
HF_TOP_K=10
HF_RERANK_TOP_N=10

ANONYMIZATION_SALT=change_this_value
```

## HR-поиск кандидатов

HR-поиск работает в несколько этапов.

### 1. Сбор данных

Из базы данных собираются сведения о сотрудниках:

- Базовый профиль
- Образование
- Дипломы
- Опыт работы
- Навыки
- Дополнительные курсы
- Курсы повышения квалификации
- Участие в соревнованиях

### 2. Построение search-документов

Для каждого сотрудника формируется нормализованный документ:

- Текст профиля
- Текст навыков
- Текст опыта
- Текст образования
- Текст курсов
- Агрегированный текст
- Структурированный payload

### 3. Анонимизация

Для HR-поиска кандидат отображается через анонимный код вида:

```text
CAND-XXXXXXXXXX
```

В анонимный профиль не должны попадать:

- ФИО
- Серия и номер диплома
- Имена файлов
- Пути к файлам
- Иные явно идентифицирующие данные

### 4. Retrieval

Требования к должности преобразуются в embedding.  
Затем рассчитывается близость между embedding требований и embedding документов кандидатов.

### 5. Ranking и reranking

После первичного поиска кандидаты ранжируются по релевантности.  
После активации reranker-модели выполняется дополнительная сортировка наиболее подходящих кандидатов.

### 6. Отправка приглашения

HR выбирает кандидата из анонимного списка и отправляет ему приглашение.  
Сотрудник получает уведомление в личном кабинете.

## Загрузка файлов

Сотрудник может прикладывать файл к заявке на изменение резюме.

Поддерживаемые форматы:

- PDF;
- DOC;
- DOCX;
- JPG;
- JPEG;
- PNG.

Ограничения:

- максимальный размер файла: 10 МБ;
- исполняемые файлы запрещены;
- файл проходит базовую проверку расширения, MIME-типа и сигнатуры;
- файл сохраняется в каталоге `uploads`.
